The MC World Compressor Backend includes a Discord webhook integration that sends real-time notifications when worlds are uploaded, processed, or encounter errors.

## Overview

Notifications are sent through Discord webhooks using a custom notification channel. The system tracks five distinct status events:

- **subido** - World file successfully uploaded
- **procesando** - World compression has started
- **listo** - World processing completed successfully
- **error** - Processing failed with an error
- **expirado** - World file has expired (24 hours after upload)

## Setting Up Discord Webhooks

<Steps>
  <Step title="Create a Discord webhook">
    In your Discord server:
    
    1. Navigate to **Server Settings** → **Integrations** → **Webhooks**
    2. Click **New Webhook**
    3. Configure the webhook:
       - Name: "MC World Compressor"
       - Channel: Select your notification channel
    4. Click **Copy Webhook URL**
  </Step>
  
  <Step title="Configure the application">
    Add the webhook URL to your environment configuration:
    
    ```bash .env
    DISCORD_NOTIFICATIONS_WEBHOOK=https://discord.com/api/webhooks/1234567890/AbCdEfGhIjKlMnOpQrStUvWxYz
    ```
    
    Or in your Docker run command:
    
    ```bash
    docker run -d \
      -e DISCORD_NOTIFICATIONS_WEBHOOK="https://discord.com/api/webhooks/..." \
      mc-world-compressor:latest
    ```
  </Step>
  
  <Step title="Verify configuration">
    The webhook URL is read from the config at runtime:
    
    ```php
    $webhookUrl = config('services.discord.notifications_webhook');
    ```
    
    Ensure your `config/services.php` includes:
    
    ```php
    'discord' => [
        'notifications_webhook' => env('DISCORD_NOTIFICATIONS_WEBHOOK'),
    ],
    ```
  </Step>
</Steps>

## Notification Implementation

The notification system consists of two main components:

### WorldStatusNotification Class

Location: `app/Notifications/WorldStatusNotification.php`

This notification class formats the Discord message with embedded rich content:

```php
public function __construct(
    $worldId, 
    string $statusType, 
    string $messageText, 
    ?string $detailsText = null
)
```

**Parameters:**
- `$worldId` - The server/world ID from the database
- `$statusType` - Event type: `subido`, `procesando`, `listo`, `error`, `expirado`
- `$messageText` - Main notification message
- `$detailsText` - Optional additional details (e.g., error messages)

### DiscordWebhookChannel Class

Location: `app/Channels/DiscordWebhookChannel.php`

Handles the actual HTTP request to Discord's API:

```php
public function send($notifiable, Notification $notification)
{
    $payload = $notification->toDiscordWebhook($notifiable);
    $webhookUrl = config('services.discord.notifications_webhook');
    
    $response = Http::post($webhookUrl, $payload);
}
```

## Embed Customization

The notification embed structure is built in `WorldStatusNotification.php:53-95`:

```php
public function toDiscordWebhook($notifiable)
{
    $worldIdentifier = ($this->worldId) ? $this->worldId : 'Desconocido';
    $title = "Actualización del Mundo";
    $color = 0x7289DA; // Discord Blurple (default)
    
    switch ($this->statusType) {
        case 'subido':
            $title = "📤 Mundo Subido: " . $worldIdentifier;
            $color = 0x3498DB;
            break;
        // ... other cases
    }
    
    $embed = [
        'title' => $title,
        'description' => $this->messageText,
        'color' => $color,
        'timestamp' => now()->toIso8601String(),
        'footer' => ['text' => config('app.name', 'MCCompressor')],
    ];
    
    if ($this->detailsText) {
        $embed['fields'][] = [
            'name' => ($this->statusType === 'error' ? 'Detalles del Error' : 'Detalles Adicionales'),
            'value' => $this->detailsText,
            'inline' => false
        ];
    }
    
    return [
        'embeds' => [$embed],
        'username' => config('app.name') . ' Notificaciones'
    ];
}
```

### Customizing Colors

Modify the color codes in the `switch` statement to match your branding:

```php
case 'listo':
    $title = "✅ Mundo Procesado: " . $worldIdentifier;
    $color = 0x00FF00; // Bright green
    break;
```

### Adding Additional Fields

You can extend the embed with more information:

```php
$embed['fields'][] = [
    'name' => 'Tiempo de Procesamiento',
    'value' => $processingTime . ' segundos',
    'inline' => true
];

$embed['fields'][] = [
    'name' => 'Chunks Eliminados',
    'value' => $removedChunks,
    'inline' => true
];
```

## Error Handling

The Discord channel includes logging for failed webhook requests:

```php
if (!$webhookUrl) {
    Log::error('Discord webhook URL not configured for notifications.');
    return;
}

$response = Http::post($webhookUrl, $payload);

if (!$response->successful()) {
    Log::error('Failed to send Discord notification.', [
        'status' => $response->status(),
        'body' => $response->body()
    ]);
}
```

<Warning>
If the webhook URL is not configured, notifications will fail silently. Check your application logs for configuration errors.
</Warning>

## Best Practices

<AccordionGroup>
  <Accordion title="Rate limiting">
    Discord webhooks have rate limits. Avoid sending notifications in tight loops:
    
    - Maximum 30 requests per 60 seconds per webhook
    - Maximum 5 requests per 2 seconds
    
    For bulk operations, consider batching notifications.
  </Accordion>
  
  <Accordion title="Queue notifications">
    For high-volume applications, queue notifications to avoid blocking:
    
    ```php
    class WorldStatusNotification extends Notification implements ShouldQueue
    {
        use Queueable;
        // ...
    }
    ```
  </Accordion>
  
  <Accordion title="Monitor webhook health">
    Periodically verify webhook connectivity:
    
    ```bash
    curl -X POST $DISCORD_WEBHOOK \
      -H "Content-Type: application/json" \
      -d '{"content": "Health check"}'
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Notifications Not Appearing

1. **Check webhook URL**: Verify the environment variable is set correctly
2. **Check Discord permissions**: Ensure the webhook has permission to post in the channel
3. **Check logs**: Look for errors in `storage/logs/laravel.log`
4. **Test webhook directly**: Use curl to send a test message

### Invalid Webhook Errors

```json
{
  "status": 404,
  "body": "{\"message\": \"Unknown Webhook\"}"
}
```

**Solution**: The webhook was deleted or the URL is incorrect. Create a new webhook and update your configuration.
