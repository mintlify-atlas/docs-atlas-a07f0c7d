The MC World Compressor Backend provides two primary endpoints for uploading Minecraft worlds: `/api/comprimir` for compression-only uploads and `/api/subir` for chunked uploads with processing.

## Supported Formats

The API accepts the following compressed world formats:

- **ZIP** - Standard ZIP archives containing world data
- **TAR.GZ** - Gzipped TAR archives

All uploaded worlds must contain a valid `level.dat` file to be recognized as a Minecraft world.

## File Size Limits

The backend is configured with generous upload limits to accommodate large world files:

```ini
upload_max_filesize = 8192M (8 GB)
post_max_size = 8192M (8 GB)
max_execution_time = 1800 (30 minutes)
memory_limit = 8192M (8 GB)
```

<Note>
These limits are defined in the Dockerfile at `Dockerfile:38-42` and can be adjusted for your deployment needs.
</Note>

## Using the Compression Endpoint

The `/api/comprimir` endpoint provides immediate compression and optimization of world files using the Thanos library.

### Basic Upload Example

<CodeGroup>

```bash cURL
curl -X POST https://your-domain.com/api/comprimir \
  -F "mundo_comprimido=@/path/to/world.zip" \
  -H "Accept: application/json"
```

```javascript JavaScript
const formData = new FormData();
formData.append('mundo_comprimido', worldFile);

const response = await fetch('https://your-domain.com/api/comprimir', {
  method: 'POST',
  body: formData,
  headers: {
    'Accept': 'application/json'
  }
});

const result = await response.json();
console.log('Download URL:', result.download_url);
```

```python Python
import requests

with open('/path/to/world.zip', 'rb') as f:
    files = {'mundo_comprimido': f}
    response = requests.post(
        'https://your-domain.com/api/comprimir',
        files=files
    )
    
result = response.json()
print(f"Download URL: {result['download_url']}")
```

</CodeGroup>

### Response Format

```json
{
  "download_url": "/storage/compressed_worlds/optimized_my-world_abc1234567.zip"
}
```

The optimized world will be available immediately at the returned URL.

## Using the Upload Endpoint

The `/api/subir` endpoint supports both standard and chunked uploads, with automatic queuing and background processing.

### Standard Upload

For smaller world files (under 2GB), you can use a standard multipart upload:

<CodeGroup>

```bash cURL
curl -X POST https://your-domain.com/api/subir \
  -F "mundo_comprimido=@/path/to/world.zip" \
  -H "Accept: application/json"
```

```javascript JavaScript
const formData = new FormData();
formData.append('mundo_comprimido', worldFile);

const response = await fetch('https://your-domain.com/api/subir', {
  method: 'POST',
  body: formData
});

const result = await response.json();
console.log('Server ID:', result.servidor_id);
console.log('Status:', result.estado);
```

</CodeGroup>

### Response Format

```json
{
  "message": "Archivo subido y procesado.",
  "servidor_id": 42,
  "estado": "pendiente"
}
```

The `servidor_id` can be used to check the processing status using the `/api/estado/{id}` endpoint.

## Checking Upload Status

After uploading via `/api/subir`, you can check the processing status:

```bash cURL
curl https://your-domain.com/api/estado/42
```

## World File Structure

The uploaded archive must contain a valid Minecraft world structure. The API will search for `level.dat` in:

1. **Root directory** - `level.dat` directly in the archive root
2. **Single subdirectory** - One folder containing `level.dat`
3. **First-level subdirectories** - Multiple folders, one containing `level.dat`

<Warning>
If no `level.dat` file is found, the upload will fail with an error message.
</Warning>

## Error Handling

Common error responses:

```json
{
  "error": "No se recibió ningún archivo ZIP. Asegúrese de que el campo del formulario se llame 'mundo_comprimido'."
}
```

```json
{
  "error": "No se pudo encontrar el directorio del mundo (que contenga level.dat) dentro del ZIP."
}
```

<Tip>
Always use the field name `mundo_comprimido` when uploading files to either endpoint.
</Tip>
