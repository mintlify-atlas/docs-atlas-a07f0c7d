## Overview

MC World Compressor provides a simple REST API to compress Minecraft worlds using the powerful Thanos library. This guide will walk you through uploading a world, processing it, and downloading the compressed result.

<Note>
The API supports both single-file uploads and chunked uploads for large world files.
</Note>

## Quick Start

<Steps>
  <Step title="Upload Your Minecraft World">
    Upload your world as a ZIP file to the `/api/subir` endpoint:

    ```bash
    curl -X POST http://localhost:8000/api/subir \
      -F "mundo_comprimido=@/path/to/your/world.zip"
    ```

    **Response:**
    ```json
    {
      "message": "Archivo subido y procesado.",
      "servidor_id": 42,
      "estado": "pendiente"
    }
    ```

    <Info>
    Save the `servidor_id` - you'll need it to check the processing status and download your compressed world.
    </Info>
  </Step>

  <Step title="Check Processing Status">
    Use the `servidor_id` from the upload response to check the status:

    ```bash
    curl http://localhost:8000/api/estado/42
    ```

    **While processing:**
    ```json
    {
      "estado": "pendiente",
      "cola": "1/3"
    }
    ```

    **When ready:**
    ```json
    {
      "estado": "listo",
      "download_url": "/storage/compressed_worlds/optimized-world_abc123.zip",
      "fecha_expiracion": "2026-03-06T12:00:00.000000Z",
      "tamano_inicio": 125.5,
      "tamano_final": 45.2
    }
    ```

    <Tip>
    The world is automatically compressed by removing unused chunks with zero inhabited time.
    </Tip>
  </Step>

  <Step title="Download Your Compressed World">
    Once the status shows `"estado": "listo"`, download your compressed world:

    ```bash
    curl -O http://localhost:8000/storage/compressed_worlds/optimized-world_abc123.zip
    ```

    <Warning>
    Compressed worlds expire after 24 hours. Make sure to download your world before the `fecha_expiracion` timestamp.
    </Warning>
  </Step>
</Steps>

## Alternative: Direct Compression Endpoint

For simpler use cases, you can use the direct compression endpoint that processes and returns the download URL immediately:

```bash
curl -X POST http://localhost:8000/api/comprimir \
  -F "mundo_comprimido=@/path/to/your/world.zip"
```

**Response:**
```json
{
  "download_url": "/storage/compressed_worlds/optimized_my-world_xyz789.zip"
}
```

<Note>
This endpoint processes the world synchronously, so it may take longer for large worlds. For better queue management and progress tracking, use the `/api/subir` endpoint instead.
</Note>

## Checking Queue Status

To see how many worlds are currently being processed:

```bash
curl http://localhost:8000/api/cola
```

**Response:**
```json
{
  "cola": 5
}
```

## World Requirements

<Info>
Your Minecraft world must:
- Be packaged as a ZIP file
- Contain a `level.dat` file at the root or in a single subdirectory
- Be a valid Anvil-format world (Minecraft Java Edition)
</Info>
